## File Organization Rules

### What Goes Where

| Type | Location | Example |
|------|----------|---------|
| HTML content | `/` | `index.html` |
| CSS styling | `/css` | `style.css` |
| JavaScript logic | `/js` | `app.js` |
| Video frames | `/frames` | `frame_0001.webp` |
| Documentation | `/docs` + root | `SKILL.md`, `README.md` |
| Assets | `/assets` | fonts, images, videos |
| Config files | `/` | `.env`, `.gitignore`, `package.json` |
| Skills | `/.claude/skills` | workflow guides |
| Build output | `/public` | (if using build process) |

### What NOT to Do

вќЊ **Don't** create new top-level CSS/JS files вЂ” keep to `/css` and `/js`
вќЊ **Don't** scatter documentation everywhere вЂ” use `/docs` or root only
вќЊ **Don't** commit large assets вЂ” use `/assets` + `.gitignore`
вќЊ **Don't** mix HTML fragments into JS вЂ” keep structure in HTML
вќЊ **Don't** nest deeply вЂ” max 3 levels deep (cleaner browsing)
